## Зачем проводить учения

DRP без регулярных учений — это инструкция по плаванию, которую читали, но в воду не заходили. Учения решают три задачи:

1. **Проверяют актуальность плана** — инфраструктура меняется, план должен за ней успевать
2. **Проверяют работоспособность бэкапов** — резервная копия, которую не восстанавливали, не является резервной копией
3. **Обучают команду** — в реальном инциденте нет времени читать инструкции впервые

## Типы учений

### Тип 1. Проверка резервных копий
*Ежемесячно · 1–2 часа · Технический специалист*

Минимальный обязательный тест. Берём случайный файл из бэкапа файлового сервера и базу 1С — восстанавливаем и проверяем читаемость. Фиксируем результат в журнале.

**Чеклист:**
- [ ] Выбрать случайную дату бэкапа (не последнего)
- [ ] Восстановить 3–5 случайных файлов разных типов
- [ ] Восстановить БД 1С в тестовую среду, зайти и проверить данные
- [ ] Зафиксировать время восстановления и результат

### Тип 2. Настольные учения
*Раз в 6 месяцев · 2–3 часа · Вся команда восстановления*

Команда собирается и «проигрывает» сценарий инцидента устно. Ведущий описывает ситуацию поэтапно, участники объясняют свои действия. Реальных действий с системами не производится.

**Сценарии для обсуждения:**
- «В 3 часа ночи в пятницу зашифрованы все серверы. Что делаете?»
- «Главный системный администратор недоступен. Кто берёт руководство?»
- «Бэкап оказался зашифрован тоже. Какой следующий шаг?»
- «Инцидент произошёл в период закрытия квартала. Как расставляете приоритеты?»

**Результат:** список расхождений между планом и реальными намерениями команды. Обновление плана.

### Тип 3. Частичные учения
*Раз в 6 месяцев · 4–8 часов · Технические группы*

Восстановление одной системы в изолированной тестовой среде. Цель — проверить работоспособность конкретной процедуры из DRP.

**Примеры сценариев:**
- Полное восстановление AD из System State бэкапа на чистой ВМ
- Развёртывание сервера 1С из бэкапа SQL
- Восстановление конфигурации MikroTik на тестовом устройстве

### Тип 4. Полные учения
*Раз в год · Выходной день · Весь персонал DRP*

Симуляция полного отказа инфраструктуры. Производственные системы не затрагиваются — восстановление ведётся в тестовой среде или на резервном оборудовании. Измеряется реальный RTO.
