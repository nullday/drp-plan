## РАЗДЕЛ 1. ВВЕДЕНИЕ

### 1.1 Назначение

Настоящий DRP является обязательным к исполнению операционным документом, определяющим процедуры, роли и технические меры, необходимые для восстановления IT-инфраструктуры организации после любого типа катастрофического события. Документ должен быть доступен в любое время, в том числе при отсутствии доступа к корпоративным системам.

### 1.2 Принципы документа

- **Исполнимость:** каждая процедура должна быть выполнима конкретным сотрудником без дополнительных уточнений
- **Актуальность:** документ пересматривается не реже раза в 6 месяцев и при каждом существенном изменении инфраструктуры
- **Доступность:** физическая копия хранится в трёх местах: серверная, кабинет директора, внешнее хранилище

### 1.3 Связанные документы

| Документ | Версия | Расположение |
|---|---|---|
| Политика информационной безопасности | `[v.x]` | `[путь]` |

## РАЗДЕЛ 6. ПРОЦЕДУРЫ ВОССТАНОВЛЕНИЯ

### 6.1 Уровни объявления инцидента

| Уровень | Триггер | Кто объявляет | Состав реагирования |
|---|---|---|---|
| **ALPHA** (учения) | Плановые учения | Руководитель IT | Технические группы |
| **BRAVO** (значительный) | Деградация ≥ 2 Tier-2 систем | Руководитель IT | Группы А, Б или В |
| **CHARLIE** (критический) | Недоступна ≥ 1 Tier-1 система | Директор IT | Все группы |
| **DELTA** (катастрофа) | Полная недоступность инфраструктуры | Генеральный директор | Кризисный штаб + все группы |

### 6.2 Процедура DELTA: Полное восстановление

#### Первые 30 минут — Активация

- [ ] Директор IT объявляет уровень DELTA
- [ ] Активирован кризисный штаб: телефонная конференция / физический сбор в `[место]`
- [ ] Каждая группа получает свои задачи по данному плану
- [ ] Определён командный центр: `[физическое место / Teams / иное]`
- [ ] Активирован запасной канал связи: `[резервные SIM / мессенджер]`

#### 30 мин – 2 часа — Сдерживание и оценка

**Все группы параллельно:**

| Группа | Действие |
|---|---|
| А (Сеть) | Изоляция заражённых сегментов, сохранение журналов файрвола |
| Б (Серверы) | Инвентаризация повреждённых систем, проверка состояния бэкапов |
| В (Приложения) | Перечень недоступных приложений, приоритеты восстановления |
| Г (Места) | Опрос сотрудников, сбор информации о заражённых ПК |
| Штаб | Коммуникации с руководством, оценка ущерба, решение о внешних ресурсах |

#### 2–24 часа — Восстановление Tier-1

**Группа А: Сетевая инфраструктура**
```
□ Полный сброс пограничного маршрутизатора
□ Восстановление конфигурации из бэкапа / настройка с нуля
□ Восстановление VLAN-структуры
□ Закрытие внешнего RDP и всех нестандартных правил
□ Смена всех паролей сетевого оборудования
□ Проверка: связность между VLAN, интернет, VPN
□ Передача управления → Группа Б
```

**Группа Б: Active Directory**
```
□ Подготовка чистого Windows Server (физический или ВМ)
□ Восстановление первого DC из System State бэкапа
□ dcdiag /v — устранение всех ошибок
□ Восстановление второго DC (репликация или второй бэкап)
□ Двукратный сброс пароля krbtgt (интервал ≥ 10 мин)
□ Аудит Domain Admins и Enterprise Admins
□ Сброс паролей всех пользователей домена
□ Проверка: аутентификация, GPO, DNS
□ Запуск файлового сервера → Группа Б продолжает
```

**Группа В: Базы данных и 1С**
```
□ Установка MS SQL Server на чистую систему
□ Восстановление баз 1С из .bak бэкапа
□ Проверка целостности всех БД (DBCC CHECKDB)
□ Установка сервера 1С, настройка кластера
□ Тестовый вход в каждую ИБ, верификация данных с бухгалтерией
□ Настройка автоматического бэкапа SQL
□ Запуск остальных бизнес-приложений по приоритету
```

#### 24–72 часа — Восстановление Tier-2 и Tier-3

**Параллельно:**
- Группа Г: Массовое восстановление рабочих мест (приоритет по отделам)
- Группа В: Почта, CRM, остальные приложения
- Группа А: Телефония, СКУД, видеонаблюдение

**Приоритет отделов при восстановлении рабочих мест:**

| Очередь | Отдел | Кол-во мест | Обоснование |
|---|---|---|---|
| 1 | Бухгалтерия | `[N]` | Операционные платежи, 1С |
| 2 | Руководство | `[N]` | Принятие решений |
| 3 | `[Ключевой отдел]` | `[N]` | `[Обоснование]` |
| 4 | `[Остальные]` | `[N]` | По мере готовности |
